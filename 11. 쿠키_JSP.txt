1. 쿠키(Cookie)
  1) 클라이언트의 상태 정보를 클라이언트의 PC에 저장한다.
  2) 기본 동작
  3) 응용
  
2. 쿠키란?
  1) 클라이언트의 상태 정보를 유지하기 위한 기술이다.
  2) 주로 웹 브라우저에 키와 값 형태로 저장했다가 다음 요청 시 저장된 쿠키를 함께 전송한다.
  3) 제약 - 3000개까지 만들 수 있다.
          - 쿠키 하나의 최대 크기는 4096바이트
          - 하나의 호스트나 도메인에서 최대 50개까지 만들 수 있다.
  4) 동작 메커니즘
    - 1. 클라이언트가 서버에 요청을 보냄
    - 2. 서버가 쿠키를 생성하여 HTTP 응답 헤더에 실어 클라이언트에 전송
    - 3. 클아이언트는 쿠키를 받아 저장해둔다.
        - 서버는 아직 쿠키를 사용할 수 없다.
    - 4. 클라이언트가 다음 번 요청 시 저장해둔 쿠키를 HTTP 요청 헤더에 실어 보낸다.
    - 5.  서버는 쿠키의 정보를 읽어 필요한 작업 수행한다.
  5) 쿠키의 속성, API
    - 이름 : 쿠키를 구별하는 이름이다.
    - 값 : 쿠키에 저장할 실제 데이터
    - 도메인 : 쿠키를 적용할 도메인
    - 경로 : 쿠키를 적용할 경로
    - 유지 기간 : 쿠키를 유지할 기간
    - void setValue(String value) : 쿠키 값 설정, 쉼표, 세미콜론 같은 문자는 포함 불가다.
    - void setDomain(String domain) 
    - void setPath(String path) 
    - void set MaxAge(int seconds) : 초 단위 설정, 기간 설정하지 않으면 브라우저가 닫힐 때 삭제된다.
